# PanDownload 界面复活版 (2026纪念版)

这是一个经过修改的 PanDownload 版本，旨在**复活其用户界面**，使其能够绕过启动报错并正常进入主程序。

**⚠️ 注意：仅界面可用，核心功能（登录、下载）因百度接口升级已彻底失效。**

## 📂 目录结构

- **`Start.vbs`**：一键启动脚本（推荐）。
- **`pdproxy_bypass.py`**：Python 代理脚本，用于拦截启动验证。
- **`PanDownload/`**：PanDownload 主程序文件。

## 🚀 如何启动

1.  双击运行 **`Start.vbs`**。
2.  等待约 2-3 秒。
3.  PanDownload 主界面将自动打开（全程无黑框）。

## 🔧 实现原理

1.  **拦截启动请求**：
    PanDownload 启动时会强制检查更新并连接 `pandownload.com`。由于原服务器已挂，会导致软件卡死或报错。
    本项目使用 `pdproxy_bypass.py` 在本地 `8888` 端口开启一个微型 HTTP 服务器，劫持并伪造了 `api/init` 等接口的响应，欺骗 PanDownload 认为服务器正常，从而成功进入界面。

2.  **静默启动**：
    使用 VBScript 脚本调用系统 Python 环境（`python.exe`）运行代理脚本，并强制隐藏控制台窗口，实现无感启动。

## 🚫 已知限制

*   **无法登录**：尝试登录会提示“账号登录失败”或 `0x38` 错误。这是因为百度网盘现在的登录接口强制校验 HTTPS 签名和参数，而 PanDownload 内置的请求逻辑已过时，且无法通过简单代理修复。
*   **无法下载**：同理，文件获取和下载功能均无法使用。
*   **无法管理账号**：虽然能进界面，但无法与百度服务器通信，因此无法进行任何云端操作。

## 📘 学习与替代方案

如果你对 PanDownload 的技术原理或复刻感兴趣，建议关注以下方向：

1.  **BaiduPCS-Go（Golang）**
    *   开源的百度网盘命令行客户端，维护活跃。
    *   学习重点：模拟登录流程、列目录/取文件信息、分片下载与并发、UA/Headers/Cookies 的伪装与复用。

2.  **油猴脚本（Tampermonkey）**
    *   在 GreasyFork 等平台搜索“百度网盘直链”。
    *   学习重点：如何在浏览器环境获取直链与鉴权参数、通用的 HTTP 请求头/Cookie/查询参数组织方式。

3.  **PanDownload 的 UI（Duilib）**
    *   如果你喜欢的是 PanDownload 的界面，建议学习 C++ 与 Duilib（或 Qt/Electron 等现代框架）。
    *   说明：原版 UI 与逻辑封装在 `PanDownload.exe` 中，闭源且不可直接复用；可参考其交互样式，自主实现前端与后端。
